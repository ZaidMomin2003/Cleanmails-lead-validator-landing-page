Documentation

Complete guide to installing, configuring, and using CleanMails.

Installation

CleanMails installs on any Linux VPS with a single command. Requirements: Ubuntu 22.04+, 1GB RAM, port 25 open for SMTP verification.

Quick Install

After purchasing, you'll receive a license key. Run this on your VPS:

$ curl -sSL https://install.cleanmails.online/setup.sh | bash -s -- --key=CM_LIVE_xxxxxxxxxxxx

The installer will:

  • → Validate your license key
  • → Install Docker + Docker Compose (if not present)
  • → Set up PostgreSQL 16 + Redis 7
  • → Deploy the CleanMails application
  • → Configure Nginx with your domain
  • → Auto-provision SSL via Let's Encrypt
  • → Create your admin account

Manual Install (Docker Compose)

$ git clone https://github.com/cleanmails/cleanmails.git $ cd cleanmails $ cp .env.example .env $ # Edit .env with your settings $ docker compose up -d

VPS Requirements

OS: Ubuntu 22.04+ / Debian 12+
RAM: 1GB min, 2GB recommended
CPU: 1 core min, 2 recommended
Disk: 20GB SSD
Ports: 80, 443, 25 (SMTP)
Cost: ~$5–10/month VPS

Configuration

All settings are managed via environment variables in the .env file.

PORT=8080 DATABASE_URL=postgres://cleanmails:password@localhost:5432/cleanmails?sslmode=disable REDIS_URL=redis://localhost:6379 JWT_SECRET=your-random-64-char-secret ENVIRONMENT=production ALLOWED_ORIGIN=https://yourdomain.com DODO_API_KEY=your-dodo-api-key DODO_PRODUCT_ID=your-product-id
Important: In production, JWT_SECRET must be a strong random string. The server will refuse to start if it detects a weak secret.

First Verification

After installation, open your domain in a browser. Log in with the admin credentials you created during setup.

  1. 1 Navigate to the Verify tab
  2. 2 Enter an email address (e.g. test@gmail.com)
  3. 3 Click Verify — results appear in under 2 seconds

Bulk Upload

Upload CSV files with up to 100,000 emails. CleanMails uses a two-pass verification system:

Pass 1 — Fast Elimination

Runs instantly on all emails. Removes obvious invalids:

  • • Invalid syntax (bad format)
  • • Disposable domains (123,000+ in database)
  • • Domains with no MX records

Pass 2 — SMTP Verification

Only runs on emails that survived Pass 1. Groups by domain for efficiency:

  • • Connects to mail server via port 25
  • • Sends EHLO, MAIL FROM commands
  • • Tests for catch-all (triple-probe)
  • • Verifies each email with RCPT TO

CSV Handling

CleanMails preserves all original columns. It auto-detects the email column and appends two new columns to the output:

# Input CSV name,email,company John,john@gmail.com,Acme # Output CSV (original columns preserved + 2 new) name,email,company,cm_status,cm_reason John,john@gmail.com,Acme,valid,Mailbox confirmed via SMTP

Filtered Downloads

Download specific subsets of your results:

GET /api/v1/lists/:id/download?filter=all # All emails GET /api/v1/lists/:id/download?filter=valid # Only good emails GET /api/v1/lists/:id/download?filter=invalid # Only bad emails GET /api/v1/lists/:id/download?filter=risky # Disposable/catch-all

Verification Checks

CleanMails performs 8 distinct checks on every email:

spellcheck

Syntax Validation

RFC-compliant regex check. Catches typos like missing @, double dots, invalid characters.

dns

MX Record Lookup

Verifies the domain has mail exchange records configured. No MX = can't receive email.

handshake

SMTP Handshake

Full protocol conversation: EHLO → MAIL FROM → RCPT TO. Confirms the mailbox exists at the server level.

delete_forever

Disposable Detection

123,000+ throwaway domains (Mailinator, Guerrillamail, etc.). Updated daily from open-source feeds.

select_all

Catch-All Detection

Triple-probe algorithm sends 3 fake addresses. If 2+ accepted, domain is catch-all — individual verification unreliable.

badge

Role Account Detection

890+ patterns: admin@, info@, support@, noreply@. These are shared inboxes, not personal addresses.

language

Free Domain Detection

4,400+ providers: Gmail, Yahoo, Hotmail, Outlook. Flags personal vs business addresses.

lightbulb

Typo Suggestion

Levenshtein distance matching: gmial.com → gmail.com, yaho.com → yahoo.com.

Smart Catch-All Detection

A catch-all domain accepts emails to any address — making individual RCPT TO verification unreliable. CleanMails uses a triple-probe approach for accuracy:

# CleanMails sends 3 guaranteed-fake addresses: RCPT TO: cm_probe_1719238471_a@domain.com → 250 OK RCPT TO: cm_verify_1719238472_x@domain.com → 250 OK RCPT TO: zz__nonexist_99182@domain.com → 250 OK # 2+ accepted = catch-all confirmed # All emails on this domain marked as "unknown (catch-all)" # No further RCPT TO attempts — saves connections

Results are cached per domain. If you have 5,000 emails on a catch-all domain, only 3 probe requests are made — not 5,000.

MCP Server

CleanMails includes a Model Context Protocol server. AI agents (Claude, Cursor, Kiro) can verify emails through natural language.

Setup

Add to your IDE's mcp.json:

{ "mcpServers": { "cleanmails": { "command": "cleanmails-mcp", "args": ["--api-url", "http://localhost:8080"], "env": { "CLEANMAILS_API_KEY": "cm_live_..." } } } }

Webhooks

Receive HTTP notifications when events occur. Configure endpoints from the Developer page or via API.

Events

  • verification.completed — Single email verified
  • list.completed — Bulk list processing finished
  • email.risky — Disposable or catch-all detected

Payload Format

// POST to your webhook URL { "event": "verification.completed", "timestamp": "2025-06-17T10:30:00Z", "data": { "email": "user@company.com", "status": "valid", "score": 95, "reason": "Mailbox confirmed via SMTP" } } // Auth header included: X-CleanMails-Signature: sha256=abc123...

Connecting to n8n

In n8n, create a Webhook node with these settings:

URL: https://yourdomain.com/api/v1/webhooks/incoming Method: POST Authentication: Header Auth Header Name: X-API-Key Header Value: cm_live_xxxx

Integrations

Connect CleanMails to your existing tools through native integrations.

HubSpot

Sync contacts → verify → update custom property with verification status. Connect via OAuth2 in the Integrations page.

Zapier

Triggers: verification.completed, list.completed. Actions: verify email, upload list. Auth via API key.

n8n

Self-hosted workflow automation. Use the HTTP Request node with X-API-Key header to call any CleanMails endpoint.

Make (Integromat)

Use the HTTP module. Set URL to your API endpoint, auth header to X-API-Key, and parse JSON response.

Mailchimp

Export your Mailchimp audience as CSV → upload to CleanMails → download clean list → re-import.

Custom HTTP

Push results to any REST endpoint. Configure URL, auth method, and which events to send via the Webhooks page.

Automation Rules

Set up rules that automatically process verification results. Rules fire on every verification.

Rule Structure

// Rule: reject disposable emails { "trigger": "on_verification", "conditions": [ { "field": "result.disposable", "operator": "is_true" } ], "actions": [ { "type": "reject" }, { "type": "webhook_fire", "params": { "webhook_id": "wh-123" } } ] }

Available Triggers

  • on_verification — fires after each email is verified
  • on_list_complete — fires when a bulk list finishes

Available Actions

  • reject — mark email as rejected
  • tag_email — add a custom tag
  • webhook_fire — trigger a specific webhook
  • move_to_list — move to a different list

Seats & Users

Create additional user accounts for team members. Each seat gets independent login credentials and access to the dashboard.

POST /api/v1/seats { "name": "Sarah Miller", "username": "sarah", "password": "securepass123" }

Users created via seats can log in, upload lists, verify emails, and view analytics. Admin can remove seats at any time.

Blacklist Monitoring

CleanMails checks your IP against 5 major blacklists (Spamhaus, Barracuda, SpamCop, SORBS, UCEPROTECT) every 60 minutes. If listed, SMTP verification pauses automatically and a notification appears in the dashboard.

Updates

CleanMails includes lifetime updates. Pull the latest version with one command:

$ cleanmails update

Other management commands:

$ cleanmails status # Check service health $ cleanmails logs # View application logs $ cleanmails backup # Database backup to /opt/cleanmails/backups/ $ cleanmails restart # Restart all services
